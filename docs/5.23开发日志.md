## 2. KV cache 接口最终设计

### 2.1 数据结构

```c
typedef struct {
    /* 配置(创建时只读) */
    uint32_t num_layers;
    uint32_t num_kv_heads;
    uint32_t max_seq;
    uint32_t head_dim;

    /* device 端 buffer */
    uint64_t K_dev_off;   /* gpuMalloc 返回值, size = num_layers * num_kv_heads * max_seq * head_dim * 4 */
    uint64_t V_dev_off;

    /* 状态机 */
    uint32_t filled;      /* 已 commit token 数, [0, max_seq] */
    uint32_t pending;     /* 本 token 已 append layer 数, [0, num_layers] */
} kv_cache_t;
```

### 2.2 调用约定(严格协议)

```c
for (uint32_t L = 0; L < num_layers; L++) {
    /* 1. 算这一层的 K_new, V_new (host 端) */
    /* 2. 写入 cache(不推进 filled,推进 pending) */
    kv_cache_append_layer(ctx, cache, L, K_new, V_new);

    /* 3. 拿 offset 和 cur_pos */
    uint64_t K_off   = kv_cache_K_off(cache, L);    /* = K_dev_off + L * num_kv_heads*max_seq*head_dim*4 */
    uint64_t V_off   = kv_cache_V_off(cache, L);
    uint32_t cur_pos = kv_cache_cur_pos(cache);     /* = filled */

    /* 4. launch attention 三件套 (cur_pos 用闭区间语义 j∈[0, cur_pos]) */
    qkt_decode_launch(...);
    softmax_decode_launch(...);
    pv_decode_launch(...);
}
kv_cache_commit_token(cache);   /* 5. filled++; pending=0; */
```

**关键不变量**:在 token 内部所有 layer 的 forward 期间,`filled` 保持不变(等于当前 token 的 index)。只有 commit 推进 filled。

### 2.3 错误码

| 函数           | 错误码 | 含义                                                   |
| -------------- | ------ | ------------------------------------------------------ |
| `append_layer` | -1     | cache 已满 (`filled >= max_seq`)                       |
| `append_layer` | -2     | layer 越界 / 顺序错 / 重复 append (`layer != pending`) |
| `append_layer` | -3     | gpuMemcpy 失败(device 拷贝错误)                        |
| `commit_token` | -1     | 未 append 任何 layer 就 commit                         |
| `commit_token` | -2     | 漏 layer (`pending != num_layers`)                     |

### 2.4 寻址公式

K_cache layout `[num_layers, num_kv_heads, max_seq, head_dim]` row-major:

```
K_off(L)        = K_dev_off + L * num_kv_heads * max_seq * head_dim * 4
write_addr(kh)  = K_off(L) + kh * max_seq * head_dim * 4 + filled * head_dim * 4
```

`append_layer` 内部每个 token 做 `2 * num_kv_heads` 次小 memcpy(K 和 V 各 num_kv_heads 次,每次 `head_dim * 4` 字节)。

## 3. softmax_decode.S 关键 bug 修复

### 3.1 Bug 现象

V20 测试矩阵(cur_pos = 31 / 15 / 99,对应 S = 32 / 16 / 100)全 PASS。
V22 Part B 密集采样 step ∈ [0, 39] 后,**S=2,4,5,6,8,9,...,30 大量 FAIL**,max_err 量级 ~1e0,而 S=1,2,4,8,16,32+ 都 PASS。

### 3.2 根本原因

```
当 S < 32 时,有 (32 - S) 个 "空 lane",它们 iter_count=0,不应处理任何有效 j。
原代码 `beqz s4, done` 让空 lane 直接跳到 done,跳过了 ft0=-INF / ft6=0 的初始化。
但 VSHFL_XOR reduce 是无条件 5 步执行,空 lane 的 ft0/ft6 是寄存器残留的脏值,
污染了 warp_max 和 warp_sum 的归约。
```

S = 2 的幂时 PASS 是脆弱巧合,不是结构正确;V20 的采样恰好命中"全 PASS 区"(S=32/100 全 lane 进循环 + S=16 是 2 幂巧合)。

### 3.3 修复

- 删除原 `beqz s4, done`(让空 lane 也执行 ft0/ft6 单位元初始化)
- Pass 1 / Pass 2 循环入口加 `beqz s5, pX_done`(0 次迭代时跳过循环体)
- Pass 3 入口加 `beqz s5, done`(Pass 3 无 reduce,空 lane 直接退出避免越界写)
- 空 lane 的 ft0=-INF / ft6=0 自然参与 reduce,fmax/fadd 单位元不影响结果

### 3.4 修复后误差

step 0~39 全 PASS,max_err ≤ 1.43e-6(<= 4 ULP),跨 lane stride=32 边界(step 31→32)平滑过渡。

## 5. 新增原则

### 原则 11:warp-wide reduce 中空 lane 必须显式初始化为操作单位元

任何无条件执行的 cross-lane reduce(VSHFL_XOR + fmax/fadd 等),其参与 lane 包括"循环 0 次"的空 lane。空 lane 的寄存器是脏值,必须在 reduce 之前用操作的单位元覆盖:

- `fmax` → -INF (`0xff800000`)
- `fadd` / 加法 → 0
- `fmul` / 乘法 → 1
- `fand` 位与 → 全 1
- `for` 位或 → 0

**关键时序**:单位元初始化必须**先于**任何"跳过空 lane"的守门跳转。错误模式:

```
beqz s4, done    ; ← 空 lane 跳走,后面的初始化也跳过了
li t0, 0xff800000
fmv.s.x ft0, t0  ; ← 空 lane 不会执行这一行
```

正确模式:先无条件初始化,再用守门跳过循环体:

```
li t0, 0xff800000
fmv.s.x ft0, t0  ; ← 所有 lane(含空)都初始化
mv s5, s4
beqz s5, loop_end  ; ← 空 lane 跳过循环体,但 ft0 已是 -INF
loop:
    ...
loop_end:
[reduce 阶段所有 lane 一起跑]
```

### 原则 12:V20 风格的 4 档采样存在覆盖盲区

V20 attention 测试 cur_pos 取值 31 / 15 / 99,对应 S = 32 / 16 / 100。这些值都落在"巧合 PASS 区":

- S ≥ 32 → 所有 lane 至少进 1 轮循环,空 lane 不存在
- S = 16 → 2 的幂,XOR shuffle 的部分巧合让脏值不显形

Part B 密集采样 step ∈ [0, N-1] 才暴露 bug。**后续 warp reduce 类 kernel 测试必须覆盖 S ∈ [3, 7, 9, 11, ..., 31] 非 2 幂档**,确保至少有 32-S 个空 lane 参与 reduce。

这条原则与 V21 原则"非重叠 lane 范围才能验证 reduce 正确"是同一族,但侧重时机:**非重叠是空间维度的非平凡,非 2 幂是数量维度的非平凡**,两者要并用。

### 原则 13:append/launch/commit 协议必须严格按序

```
append → launch → commit
```

任何顺序偏移立刻翻车:

- **append 之前调 cur_pos**:`pending == 0`,assert 拦截或返回上一轮状态
- **commit 之后调 cur_pos**:filled 已推进,kernel j 循环越界扫到未写位置
- **launch 之前 commit**:同上

`pending` 字段把"我已经 append 但未 commit"这个状态显式化,kv_cache_cur_pos 的 `assert(pending > 0)` 是该协议的运行时守门。

### 原则 14:重复的 stride 算式必须靠 getter 复用,不能复写

`kv_cache_append_layer` 内部计算写入地址时,通过 `kv_cache_K_off(cache, L)` 复用 layer base 算式,不直接展开 `K_dev_off + L * num_kv_heads * max_seq * head_dim * 4`。这避免"两处展开一处错"的偏差风险。

### 原则 15:reference 不能让 PASS 退化为"任意值都 PASS"

反例(test_argmax.c):全部 logits 填 -INF → host reference 永远返回 idx=0 → 任何"恰好返回 0"的 kernel 都通过测试。

写 reference 之前必须自问:**reference 的输出是否对所有合理 kernel 实现唯一确定?** 如果不是,数据 pattern 退化了,要换数据。

具体到 reduce 类 kernel,reference 必须:

- 输入有真实的"最大值/极值",不能全是相同值或退化值
- 数据范围跨越多个 lane,且 lane 之间不对称
- 失败时能区分"kernel 完全没跑"和"kernel 跑了但错了"

- 数据 `((i % 7) - 3) * 0.1 + 1e-5` 用 7 周期打破 lane 对称(V21 教训的正面应用)
- `fmaf` 强制单舍入匹配 device fmadd.s(数值严格匹配)
  小补充:token_pos 取 0/1/5/100,可加 high pos(200, 500)测 sin 高频饱和。head_dim 写死 64,其他 head_dim 未测(可记债务,Qwen2 暂不需要)。

### 8.3 重写优先级

**🔴 3.6.1 argmax**:本质未测,最紧急,重写后大概率暴露空 lane bug。

**🟠 3.6.2~3.6.4 silu / rmsnorm / gemm**:覆盖度不足,但当前测试至少跑过有意义的算式,优先级中等。

**🟢 3.6.5~3.6.6 rope / embedding**:小补强即可,优先级最低。

### 8.4 通用补强模式(用于 3.6.1~3.6.4)

参照 test_rope.c 的成熟模板:

```c
typedef struct {
    const char *name;
    /* shape 参数 */
    /* token / 位置 参数 */
} TestCase;

static const TestCase test_cases[] = {
    {"baseline",     /* 最简 PASS 基线 */},
    {"asymmetric",   /* 打破 lane 对称 */},
    {"non_pow2",     /* 非 2 幂尺寸 */},
    {"large_scale",  /* 真实推理规模 */},
};

/* helper 自检(如果有 helper)*/
static void helper_self_check(void) { /* assert */ }

/* host reference */
static void xxx_reference(...) { /* 严格平行 device fp 操作 */ }

/* 逐档跑 */
static int run_one_test(...) {
    /* alloc / upload / launch / D2H / compare(三指标)/ free */
}

int main(void) {
    helper_self_check();
    /* gpuInit / load kernel / 跨 case 共享资源 */
    int total_pass = 0;
    for (auto tc : cases) {
        if (!run_one_test(...)) break;
        total_pass++;
    }
    /* cleanup + summary */
}
```

**关键不变量**:

1. 多档 case,且 case 之间覆盖不同 lane 模式(2 幂 / 非 2 幂 / 跨 stride)
2. 数据 pattern 必须打破 lane 间对称(`i % 7` / `i % 11` / 随 i 变化)
3. reference 不能退化(原则 15)
4. baseline(最简 case)用 bit-perfect 比对,真实 case 用 max_err + tol
5. 失败时输出足够诊断信息(头 N 个样本 + bit pattern)
