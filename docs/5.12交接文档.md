## 2. decode attention 三件套最终设计

### 2.1 数据流

```
输入: Q[num_q_heads, head_dim]
      K_cache[num_kv_heads, max_seq, head_dim]
      V_cache[num_kv_heads, max_seq, head_dim]
      cur_pos, max_seq, head_dim, q_per_kv, scale

qkt_decode   → scores[num_q_heads, max_seq]   (QK^T * scale)
softmax_decode → scores[num_q_heads, max_seq]  (in-place softmax, 即P)
pv_decode    → O[num_q_heads, head_dim]        (P * V)
```

### 2.2 各 kernel args 结构体

```c
struct qkt_decode_args {
    uint32_t Q_off;        /* [num_q_heads, head_dim] */
    uint32_t K_cache_off;  /* [num_kv_heads, max_seq, head_dim] */
    uint32_t scores_off;   /* [num_q_heads, max_seq] */
    uint32_t cur_pos;
    uint32_t max_seq;
    uint32_t head_dim;
    uint32_t q_per_kv;
    uint32_t scale_bits;   /* float bit pattern */
};

struct softmax_decode_args {
    uint32_t scores_off;   /* in-place buffer */
    uint32_t cur_pos;
    uint32_t max_seq;
};

struct pv_decode_args {
    uint32_t P_off;        /* softmax输出,与scores_off同一buffer */
    uint32_t V_cache_off;  /* [num_kv_heads, max_seq, head_dim] */
    uint32_t O_off;        /* [num_q_heads, head_dim] */
    uint32_t cur_pos;
    uint32_t max_seq;
    uint32_t head_dim;
    uint32_t q_per_kv;
};
```

### 2.3 dispatch（三个kernel相同）

```c
grid  = {num_q_heads, 1, 1};
block = {32, 1, 1};
```

### 2.4 lane分工

| kernel | lane分工 | reduce |
|---|---|---|
| qkt_decode | stride=32沿j，每lane算head_dim个k的点积 | 无跨lane |
| softmax_decode | stride=32沿j，三遍扫(max/exp+sum/div) | 2次XOR shuffle（max+sum） |
| pv_decode | 沿k切分(head_dim/32个k/lane)，每lane内j串行 | 无跨lane |

## 3. 关键实现细节

### 3.1 softmax_decode.S 结构

```
Pass1: local_max = fmax over j∈[lane, lane+32, ...]
       → VSHFL_XOR 5步 → warp_max (ft5保存)

Pass2: for j: e=exp(scores[j]-warp_max), scores[j]=e, local_sum+=e
       → VSHFL_XOR 5步 → warp_sum (ft6)

Pass3: for j: scores[j] /= warp_sum
```

**-INF加载**：`li t0, 0xff800000` + `fmv.s.x ft0, t0`

**步进**：j循环步进固定128字节（32lanes × 4字节），与max_seq无关；`scores_row_base`计算时才用max_seq。

**in-place安全性**：单lane内先读后写，跨lane j不重叠，三遍扫一维不交叉。

### 3.2 pv_decode.S 寄存器分配

```
s0=P_row_base  s4=max_seq*4(步进)  s8=lane_id
s1=V_kh_base   s5=head_dim         s9=kh
s2=O_row_base  s6=q_per_kv         s10=S
s3=head_dim*4  s7=qh(block_id)     s11=n_k=head_dim/32
```

## 5. 新增原则

**原则：j循环步进由warp宽度决定，不由max_seq决定**

stride=32沿j，地址步进=32×4=128字节，是warp宽度的固定函数。max_seq只影响行基址计算。两者职责分离，混淆会导致错误的步进值。

**原则：max_err=0是结构性预测，不是巧合**

无跨lane reduce的kernel（qkt、pv）→ 单lane内累加顺序与host完全一致 → max_err=0。有跨lane reduce的kernel（softmax）→ XOR shuffle改变累加顺序 → 1~4 ULP误差。判断误差是否合理靠结构预测，不靠经验。
